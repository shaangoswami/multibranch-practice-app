What is Jenkins?

Jenkins is an open-source automation server that helps teams build, test, and deploy applications continuously. It's the heart of CI/CD (Continuous Integration/Continuous Deployment) pipelines.

Jenkins Pipeline

A pipeline is a series of automated steps that code goes through from development to production. It typically includes stages like: Build â†’ Test â†’ Deploy â†’ Monitor.

Key Jenkins Terms

Job: A runnable task in Jenkins
Build: One execution of a job
Workspace: Directory where builds happen
Node: Machine where Jenkins runs jobs
Plugin: Extension that adds functionality

Jenkinsfile

A text file that defines your pipeline as code. It can be written in Declarative or Scripted syntax and is stored alongside your application code.

CI/CD Explained

Continuous Integration (CI): Automatically test code changes when committed
Continuous Deployment (CD): Automatically deploy tested code to production
Continuous Delivery: Ready for production but manual approval needed

Pipeline Stages

Build Stage: Compile code and create artifacts
Test Stage: Run unit, integration, and acceptance tests
Deploy Stage: Move code to staging or production environment
Post-actions: Cleanup, notifications, and reporting

Declarative vs Scripted Pipelines

Declarative: Structured, easier to read, recommended for most use cases
Scripted: More flexible, based on Groovy, for complex workflows

Common Jenkins Plugins

Git Plugin: Source code management integration
Docker Plugin: Build and run containers
Kubernetes Plugin: Deploy to Kubernetes clusters
Pipeline: Create sophisticated pipelines
Blue Ocean: Modern UI for pipelines

Webhooks & Triggers

Poll SCM: Jenkins periodically checks for code changes
Webhook: Git repository triggers Jenkins when changes are pushed
Manual Trigger: Run builds manually from Jenkins UI
Scheduled Builds: Run builds at specific times using cron syntax

Distributed Builds

Jenkins can run builds on multiple machines (agents/nodes) to distribute workload. A master node coordinates builds that run on agent nodes, enabling parallel execution and better resource utilization.

ğŸ“– Real Jenkinsfile Examples

Node.js Application

pipeline {
    agent any
    stages {
        stage('Install') {
            steps {
                sh 'npm install'
            }
        }
        stage('Test') {
            steps {
                sh 'npm test'
            }
        }
        stage('Build') {
            steps {
                sh 'npm run build'
            }
        }
    }
}

Python Application

pipeline {
    agent any
    stages {
        stage('Setup') {
            steps {
                sh 'pip install -r requirements.txt'
            }
        }
        stage('Lint') {
            steps {
                sh 'pylint src/'
            }
        }
        stage('Test') {
            steps {
                sh 'pytest'
            }
        }
    }
}

Docker Build & Push

pipeline {
    agent any
    environment {
        REGISTRY = 'docker.io'
    }
    stages {
        stage('Build') {
            steps {
                sh 'docker build -t myapp:${BUILD_NUMBER} .'
            }
        }
        stage('Push') {
            steps {
                sh 'docker push myapp:${BUILD_NUMBER}'
            }
        }
    }
}
